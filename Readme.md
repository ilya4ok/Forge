# Personal Dashboard — Спецификация проекта

Персональный дашборд Ильи для отслеживания обучения по нескольким трекам, управления задачами и расписанием, с AI-ассистентом на базе Claude.

> **Контекст:** «Мак» — профессиональная работа Ильи (работает удалённо за Mac). Mac-дни = дни с рабочими часами. В эти дни утреннее окно для учёбы ограничено.

---

| Технология | Назначение |
|-----------|-----------|

## Страницы

### `/` — Dashboard
- Приветствие с датой
- 3 карточки статистики: стрик / общий XP / рекорд стрика
- Список задач на сегодня — можно отметить выполненной или пропустить прямо здесь
- Прогресс по трекам (XP каждого трека, бары относительно максимального)
- Ссылка на чат
- Онбординг-экран при первом запуске (до настройки расписания)

### `/chat` — AI Чат
Основной интерфейс управления. AI получает контекст:
- Задачи на сегодня (с ID для управления)
- Рабочие часы в маке (ближайшие 7 записей)
- Стрик и XP по трекам
- Ближайшие незавершённые учебные задачи (7 штук)
- Количество настроенных учебных дней

Через чат можно **реально изменять данные** — AI возвращает actions-блок, который выполняется на клиенте. Большинство операций доступны только через чат (изменение расписания, отмена выполнения и т.д.).

### `/schedule` — Расписание
Текущая неделя, 7 карточек по дням. Каждая карточка:
- Мини-полоска с точками выполненных задач по дням недели
- Время подъёма (09:00), утренний блок (готовка/завтрак/сборы 1.5ч)
- Задачи с расчётным временем начала
- Блок работы в маке (если есть)
- Время прихода домой
- Ритуал перед сном + время отбоя

### `/tasks` — Задачи
Все задачи из store, сгруппированные по трекам. Фильтрация по треку и статусу (все / активные / выполненные / пропущенные).

### `/stats` — Статистика
Визуализация накопленного XP по трекам за всё время. Прогресс-бары и сравнение активности.

## Типы данных

```ts
type Track = 'ai' | 'design' | 'selfdevelopment' | 'mediabuy' | 'english' | 'polish' | 'gym'

type Task = {
  id: string
  title: string
  track: Track
  date: string          // YYYY-MM-DD
  completed: boolean
  skipped: boolean
  isRecurring: boolean
  recurringType?: string  // 'daily' | 'homework' | 'tutor' | 'long' | 'short'
  xp: number
}

type DayJob = {
  date: string   // YYYY-MM-DD
  start: string  // "13:00"
  end: string    // "22:30"
  label?: string
}

type StreakState = {
  current: number
  longest: number
  lastActiveDate: string
  freezeUsedMonth: string | null  // месяц использования freeze (1 раз в месяц)
}

type AppState = {
  tasks: Task[]
  workDays: string[]       // учебные дни — ВНИМАНИЕ: название противоречит интуиции,
                           // это НЕ рабочие дни, а дни когда Илья учится (YYYY-MM-DD)
  dayJobs: DayJob[]        // рабочие часы в маке по датам
  streak: StreakState
  trackXP: Record<Track, number>
  onboardingDone: boolean
  chatHistory: { role: 'user' | 'assistant'; content: string }[]
}
```

## Zustand Store

Ключ localStorage: `personal-dashboard-storage`

| Метод | Описание |
|-------|---------|
| `completeTask(id)` | Отмечает выполненной + начисляет XP + обновляет стрик |
| `uncompleteTask(id)` | Отменяет выполнение + вычитает XP |
| `skipTask(id)` | Помечает пропущенной (нейтрально для стрика) |
| `deleteTask(id)` | Удаляет задачу |
| `addTask(data)` | Добавляет разовую задачу |

| `setDayJobs(jobs)` | Устанавливает рабочие часы + регенерирует задачи месяца |
| `deleteRecurringSeries(track)` | Удаляет все повторяющиеся задачи трека (внутренний, нет AI action) |
| `addChatMessage(msg)` | Добавляет сообщение в историю чата |
| `clearChatHistory()` | Очищает историю чата |
| `processOnOpen()` | Пересчитывает стрик при открытии приложения |

Утреннее окно ~95 мин при работе с 13:00 (подъём 09:00 + сборы 90 мин = старт 10:30, выезд ~12:05). При другом времени начала работы окно меняется пропорционально.

| Польский — 30 мин | пн–пт | в потоке утра |
| Зал | пн, ср, пт | 18:00 (фиксировано) |

## Логика расписания (schedule/page.tsx)

### Константы времени
| Константа | Значение | Описание |
|-----------|---------|---------|
| `WAKE_TIME` | 09:00 | Подъём каждый день |
| `PREP_MIN` | 90 мин | Готовка + завтрак + сборы |
| `COMMUTE_TO_WORK_MIN` | 40 мин | Дорога до работы |
| `COMMUTE_HOME_MIN` | 60 мин | Дорога домой |
| `DEPART_BUF` | 15 мин | Буфер на выход |
| `GAP` | 15 мин | Пауза между задачами |
| `RITUAL_TIME` | 00:00 | Целевое начало ритуала перед сном |
| `BED_TIME` | 01:00 | Целевой отбой |

### Порядок задач (TRACK_ORDER)
english → polish → selfdevelopment → ai → design → mediabuy → gym

### Зал
- Пн/Ср/Пт: фиксировано 18:00 (не занимает утреннее окно)
- Вт/Чт/Вс/Сб на свободных днях: вычисляется по курсору после остальных задач

### Отбой (getSleepTimes)
- Если прибытие домой **до полуночи** → ритуал 00:00, отбой 01:00 (целевое)
- Если прибытие домой **после полуночи** → ритуал = время прихода, отбой = приход + 60 мин
- Сравнение использует `normMin()`: времена до 06:00 добавляют +24ч для корректного сравнения через полночь

| design | 90 |

## Система стрика (streak.ts)

- **Рост:** первая выполненная задача дня (не пропущенная)
- **Нейтрально:** все задачи дня пропущены (`skipped: true`) — стрик не ломается и не растёт
- **Сброс:** день прошёл, ни одной выполненной задачи и не все пропущены
- **Заморозка:** 1 раз в месяц — один «забытый» день не ломает стрик
- `processOnOpen()` вызывается при открытии приложения и догоняет пропущенные дни

> **Edge case — день без задач:** если на день нет ни одной задачи (не настроен `workDays`), он считается «забытым» и использует заморозку. Стрик не ломается при первом таком дне в месяце.

> **Часовые пояса:** все даты и вычисления времени — клиентские (браузер), серверная часть не участвует в работе с датами.

## API Роут — `/api/chat`

Серверный роут Next.js (ключ Claude в `.env.local`, клиент его не видит).

**Принимает:**
```json
{ "messages": [...], "context": { ... } }
```

**Контекст включает:**
- `today` / `dayOfWeek` — текущая дата
- `todayTasks` — задачи на сегодня с ID
- `dayJobs` — рабочие часы в маке (ближайшие 14, урезается до 7 в промпте)
- `streak` — текущий/рекордный стрик
- `trackXP` — XP по трекам
- `upcomingTasks` — ближайшие 7 незавершённых задач от сегодня
- `workDaysCount` — количество настроенных учебных дней

**AI возвращает:**
```
<RESPONSE>текст ответа</RESPONSE>
<ACTIONS>[{"type":"..."}]</ACTIONS>
```

**Доступные actions:**

| Тип | Параметры | Описание |
|-----|---------|---------|
| `updateSchedule` | `month`, `workDays[]` | Установить ВСЕ дни месяца (28–31 дата) |
| `setDayJobs` | `jobs[]` | Рабочие часы в маке по датам |
| `addTask` | `title`, `track`, `date`, `xp` | Добавить разовую задачу |
| `completeTask` | `taskId` | Отметить выполненной |
| `uncompleteTask` | `taskId` | Отменить выполнение (если ошибся) |
| `skipTask` | `taskId` | Пропустить |

> **Важно:** `updateSchedule` всегда содержит ВСЕ дни месяца — система сама разделяет Mac-дни и свободные дни на основе `dayJobs`.

> **Порядок вызовов:** при настройке Mac-дней AI всегда вызывает сначала `updateSchedule` (заполняет `workDays`), потом `setDayJobs`. Если `workDays` пустые — `setDayJobs` не сгенерирует задачи.

> **История не теряется:** `updateSchedule` и `setDayJobs` сохраняют уже выполненные/пропущенные задачи при регенерации. Только незавершённые задачи пересоздаются.

## Ограничения и поведение edge cases

| Ситуация | Поведение |
|----------|----------|
| История чата | Хранится в localStorage. В API отправляется последние 30 сообщений |
| Накопление задач | Старые задачи хранятся в localStorage бессрочно (нет автоархивации) |
| Переход месяца | Задачи прошлого месяца остаются в хранилище. Для нового месяца нужно вызвать `updateSchedule` |
| Отмена выполнения | Через UI нельзя — только AI (`uncompleteTask` action) |
| Удаление Mac-дня | `setDayJobs` только добавляет/обновляет. Для очистки — AI должен передать тот же день с новыми часами |
| Польский на выходных | Не генерируется автоматически (только пн–пт). Добавить вручную через чат |
